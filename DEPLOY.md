# MEDEL Tracker — Deployment Guide

Three steps: Supabase → GitHub → Render.

---

## Step 1 — Create the Supabase database

1. Go to [supabase.com](https://supabase.com) → **New project**
   - Name: `medel-tracker`
   - Save the database password you set
   - Pick the region closest to your users

2. Go to **SQL Editor → New query**, paste this, and click **Run**:

```sql
CREATE TABLE IF NOT EXISTS records (
  id                serial   PRIMARY KEY,
  patient_name      text     NOT NULL DEFAULT '',
  dob               text     NOT NULL DEFAULT '',
  phone             text     NOT NULL DEFAULT '',
  serial            text     NOT NULL DEFAULT '',
  implant           text     NOT NULL DEFAULT '',
  issue_description text     NOT NULL DEFAULT '',
  conditions        text     NOT NULL DEFAULT '',
  skin              text[]   NOT NULL DEFAULT '{}',
  visual            text[]   NOT NULL DEFAULT '{}',
  audio             text[]   NOT NULL DEFAULT '{}',
  physical          text[]   NOT NULL DEFAULT '{}',
  accessory         text[]   NOT NULL DEFAULT '{}',
  connectivity      text[]   NOT NULL DEFAULT '{}',
  steps             text[]   NOT NULL DEFAULT '{}',
  resolved          text     NOT NULL DEFAULT '',
  resolved_how      text     NOT NULL DEFAULT '',
  next_action       text     NOT NULL DEFAULT '',
  contact_name      text     NOT NULL DEFAULT '',
  contact_email     text     NOT NULL DEFAULT ''
);
```

3. Go to **Settings → Database → Connection string → URI tab**
   Copy the string — it looks like:
   ```
   postgresql://postgres:[YOUR-PASSWORD]@db.xxxxxxxxxxxx.supabase.co:5432/postgres
   ```
   Replace `[YOUR-PASSWORD]` with the password from step 1.
   
   > Use port **5432** (direct), not 6543 (pooler).

---

## Step 2 — Push to GitHub

```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/medel-tracker.git
git push -u origin main
```

> Make the repository **Private** — it contains patient data logic.

---

## Step 3 — Deploy on Render

1. Go to [render.com](https://render.com) → sign in with GitHub
2. Click **New → Web Service** → connect your `medel-tracker` repo
3. Render auto-detects `render.yaml`. Before deploying, set these **Environment Variables** in the Render dashboard:

| Key | Value |
|---|---|
| `DATABASE_URL` | Paste your Supabase URI from Step 1 |
| `APP_PASSWORD` | Your team login password (e.g. `MEDEL@VSB`) |

   (`SESSION_SECRET` is auto-generated by Render — leave it.)

4. Click **Deploy**

Your app will be live at `https://medel-tracker.onrender.com` (or similar).  
Share the URL and password with your team — that's all they need.

---

## Viewing data in Supabase

Go to **Table Editor → records** in your Supabase project to browse, filter, and export records directly.

Useful queries in the **SQL Editor**:

```sql
-- All unresolved cases
SELECT patient_name, implant, issue_description, next_action
FROM records WHERE resolved = 'No';

-- Count by device type
SELECT implant, COUNT(*) FROM records GROUP BY implant;
```

## Notes

- **Free tier:** Render apps sleep after 15 min of inactivity (~30 s wake-up). Upgrade to Render Starter ($7/mo) for always-on.
- **Backups:** Use the **Export CSV** button in the app regularly as an offline backup.
- **Password:** Change `APP_PASSWORD` from the default if sharing the URL publicly.
